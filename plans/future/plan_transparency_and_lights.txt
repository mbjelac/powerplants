This plan adds two rendering capabilities to the architect/shared pipeline:

  A. Transparent shapes — a shape's color may include an alpha (opacity) channel so it renders see-through.
  B. Light sources as shapes — a new command type that places a point light in the scene (optionally shown as a self-lit bulb), so shapes it faces are lit.

Background — how the pipeline works today (read before starting):
  - Shape text (one line per shape) is parsed into CreateBody objects by shared/parseCommands.ts. A CreateBody has: type, translate, rotate, scale, color (a "#rrggbb" string or null).
  - shared/applyCommands.ts loops over the commands and, per shape, does push() -> translate/rotate/scale -> dispatch to a drawXxx primitive -> pop().
  - Every primitive (shared/primitive/drawPyramid.ts, drawPrism.ts, drawSphere.ts, drawCylinder.ts, drawCone.ts, drawTorus.ts) calls p.fill(...colorToRgb(color)) before drawing. colorToRgb (shared/primitive/colorToRgb.ts) turns "#rrggbb" into an [r, g, b] tuple, defaulting to grey.
  - Lighting is set up once per frame in the draw loop of architect/src/sketch.ts (p.ambientLight(60) and one p.pointLight(...)). Lights are NOT objects; they are stateful calls that affect only geometry drawn after them, and they reset every frame.
  - The color picker UI (architect/src/editor/colorPicker.ts) writes c(#rrggbb) into the shape line and matches it with COLOR_REGEX = /c\(#[0-9a-fA-F]{6}\)/.

Key caveat for transparency in p5 WEBGL: transparent faces are NOT depth-sorted. With the current single unsorted loop in applyCommands and the default depth test, transparent shapes can wrongly occlude what is behind them and hide their own back faces. Section A accepts this limitation first and only adds a best-effort mitigation (draw transparent shapes last) at the end.

#1 Add an alpha channel to the color pipeline
- extend colorToRgb to return an alpha channel
    - change the return type to [number, number, number, number]
    - accept the 8-digit form "#rrggbbaa"; when only 6 digits are present, default alpha to 255 (fully opaque)
    - the no-color default stays grey and fully opaque: [160, 160, 160, 255]
    - first write tests in shared/primitive/colorToRgb.spec.ts covering: no color, 6-digit hex, 8-digit hex
- update every primitive to spread the 4-tuple into p.fill(...)
    - drawPyramid.ts, drawPrism.ts, drawSphere.ts, drawCylinder.ts, drawCone.ts, drawTorus.ts all already call p.fill(...colorToRgb(color)), so they pick up the 4th channel once the type widens; confirm each compiles

#2 Let the shape text and color picker carry alpha
- widen COLOR_REGEX in colorPicker.ts to accept an optional alpha pair: c(#rrggbb) or c(#rrggbbaa)
- add an opacity control next to the color input
    - a range input 0-255 (or 0-100 shown as percent)
    - on change, rewrite the c(...) token so its hex includes the alpha pair
    - when opacity is full, keep writing the 6-digit form so existing lines are unchanged
- confirm parseColor in parseCommands.ts accepts the 8-digit hex (widen its regex if it validates length)

#3 Best-effort transparency ordering
- in applyCommands.ts, split the loop into two passes: draw fully-opaque shapes first, then transparent ones (alpha < 255)
    - preserve original order within each pass
- note in a comment that this is a partial mitigation and overlapping transparent shapes may still show artifacts

#4 Add a light-source command type
- define a new body type for a light (e.g. "lit") in parseCommands.ts
    - it uses the existing translate and color fields: translate positions the light, color is the light's rgb
    - alpha is ignored for lights
- handle the light type in applyCommands.ts
    - lights must be emitted before shape geometry so they affect all shapes; collect light commands in a first pass and call p.pointLight(r, g, b, x, y, z) for each (using the same translate axis remapping the shapes use), then draw shapes
    - first write tests in shared/applyCommands.spec.ts (or the existing spec) asserting a light command results in a pointLight call at the mapped position, using a fake p5 that records calls
- optionally render a visible bulb at the light position
    - draw a small sphere at the light's location with p.emissiveMaterial(r, g, b) so it looks self-lit regardless of scene lighting, then reset the material for following shapes
    - gate this behind a toggle if it clutters the scene

#5 Editor support for placing a light
- add a way to insert a light command from the editor panel (architect/src/editor/editorPanel.ts and neighbours)
    - reuse the translate sliders and color picker for the light's position and color
    - no scale or rotate controls for a light
