#3 [done] object can be moved around
- [done] if after "pyr3" on the same line there is a "translate" command, apply it so that the object is placed to a location relative to it's starting one
    - example: "pyr3 t(-40, 50,120)"
    - "t" means translate
    - arguments in brackets may or may not be separated by spaces but have to be separated with commas
    - arguments are integers from 0 to 100
    - 0 means "do not move"
    - 100 means "move by the amount of floor-width (or height)"
    - 1st argument is moving left-right across the floor
    - 2nd argument is moving forward-backward across the floor
    - 3rd argument is moving up-down

    - parameterize tests if test runner library supports it

#4 [done] object can be rotated around it's center
- [done] if after "pyr3" on the same line there is a "rotate" command, apply it so that the object is rotated around it's center
    - example: "pyr3 r(20,-60) t(0, 10, -40)"
    - "r" means rotate
    - rotate command can be on the same line as translate command
    - all commands need to be separated by one or more spaces
    - rotate command has two parameters: angles in degrees, one in a vertical plane, one in horizontal plane
    - angles are integers from -180 to 180
    - 0 means "do not rotate"
    - 180 means "rotate by 180 degrees"
    - r(0, 90) means "rotate by 90 degrees in a vertical plane"
    - r(90, 0) means "rotate by 90 degrees in a horizontal plane"
- [done] add visual tests for rotation
    - parameterized on parameters: r(x, 0), r(0, x) and values x = -40, 40, -90, 90, -175, 175

#5 [done] object can be scaled
- [done] "scale" command looks like "s(100, 10,250)"
    - s function takes 3 parameters, each an integer from 1 to 1000
    - 100 means "do not scale" as it represents the percentage of the original size
    - 10 means "shrink 10 times"
    - 250 means "enlarge 2.5 times"
    - 0 is treated as 1
    - negative numbers are treated as 1
    - 1st parameter is scaling in right-left direction
    - 2nd parameter is scaling in forward-backward direction
    - 3rd parameter is scaling in up-down direction (vertically)
