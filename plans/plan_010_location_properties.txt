All locations on the map have properties.

Examples of properties are:
- soil fertility
- groundwater availability
- ore density
- cloud cover
- wind speed

These properties may influence a building's output.

Each location has a set value for all properties and that value is a floating point number.

#4 Building outputs are modified by location properties
- building definition in buildings.md may have location property in output function
    - function output row format "ResourceType <amount> propertyName"
        - or "ResourceType <amount>" in which case there is no modification
    - these are loaded into BuildingDefinition as outputModifiers: { resource: string, property: string }[]
    - add tests of parseBuildingDefinition to check that output property names are parsed correctly
        - if no property names are present, outputModifiers should be empty
        - for each output row with a property name, expect one element in the outputModifiers array
- when a building's output is used (either to calculate exports or connection possibilities)
    - find the outputModifier for the output resource type
    - get the value of the location property for the property name in the outputModifier
    - multiply the function's output amount by the property value
    - if no modifier, the function's output amount is used
    - add tests for calculating exports & connections for cases with output modifiers & location property values which modify outputs
